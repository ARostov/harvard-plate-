# Руководство для контрибьюторов

Спасибо за интерес к проекту Harvard Plate! Мы рады вашей помощи.

## Как помочь

### Сообщение об ошибках
- Используйте шаблон для багов
- Включайте шаги для воспроизведения
- Прикрепляйте скриншоты если возможно

### Предложение новых функций
- Используйте шаблон для feature request
- Объясните проблему, которую решает функция
- Предложите возможные реализации

### Pull Requests
1. Форкните репозиторий
2. Создайте ветку для ваших изменений
3. Сделайте коммиты
4. Создайте Pull Request

## Стандарты кода

### JavaScript
- Используйте современный JavaScript (ES6+)
- Следуйте существующему стилю кода
- Комментируйте сложную логику

### Vue Components
- Используйте Composition API
- Называйте компоненты в PascalCase
- Разделяйте логику и представление

### Стили
- Используйте CSS переменные из дизайн-системы
- Делайте компоненты адаптивными
- Проверяйте в разных браузерах

## Процесс разработки

1. Обсуждение: Перед началом работы обсудите изменения в issue
2. Разработка: Создайте ветку и реализуйте изменения
3. Тестирование: Протестируйте изменения локально
4. Ревью: Создайте PR и ожидайте ревью
5. Слияние: После утверждения изменения будут слиты

## Локальная разработка

### Установка
Клонируйте репозиторий и установите зависимости:

git clone https://github.com/ваш-username/harvard-plate.git
cd harvard-plate
npm install

### Запуск
Сервер разработки:
npm run dev

Production сборка:
npm run build

Предпросмотр сборки:
npm run preview

### Тестирование
Запуск тестов:
npm test

Проверка кода:
npm run lint

## Code of Conduct

Мы ожидаем, что все участники будут соблюдать:
- Уважение к другим
- Конструктивную критику
- Профессиональное поведение

Если у вас есть вопросы, создайте issue или свяжитесь с мейнтейнерами.

Спасибо за ваш вклад!
